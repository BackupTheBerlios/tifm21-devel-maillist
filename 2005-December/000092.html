Hmm, I see a lot of buzz generated by my humble
announcement (and it is humble, as nothing useful can
be done with the tifmxx right now and there is no
guarantee that it will be ever possible).

Now to the issues:
1. pci_intx is part of pci core in 2.6.14, but some
time before it was only present in libata. It well may
be that 2.6.13 without libata can have this symbol
missing.

2. The reason I ever started tifmxx (instead of
contributing to tifm21) is that I have no desire to
mess with all those "legal issues". I don't give a
shit, frankly. I only want the dumb device to work.

3. Contrary to what somebody might think, I'm working
on the windows driver code, not on linux one. First,
I've started this long before the linux driver emerged
and second, TI's linux driver is itself hastily ported
from the windows code.
