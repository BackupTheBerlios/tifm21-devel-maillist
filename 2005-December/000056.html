First, for those who missed the exciting news: few
days ago out friend Cristopher Baker managed to get
the binary driver from TI and it's now available from
Ian Morgan's page:
http://www.webcon.ca/~imorgan/tifm21/ti-binary-driver.tgz

Now on the SD issue. While MMC spec is wide open for
anybody to download and read, SD security extensions
spec is hidden and only available to licensees on an
NDA terms. This is similar to GSM security spec.

Now, the fact that support for this stuff is not
included in the linux driver removes legal obstacles
to the release of source code. It also precludes one
from using DRM-enabled SD cards on linux. One can
still use SD cards with DRM disabled, because they are
backward compatible with MMC.

I was also looking at dissassembly of win64 driver
(there is such thing). It is more readable than win32
(saner ABI). I'm also trying to guess an optimal
starting configuration for linux driver. I hope to
have some results in a few days.
